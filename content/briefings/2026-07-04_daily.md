## 🔥 今日深度推荐 Deep Reads

### 1. Simon Willison：用 DSPy 优化 Datasette Agent 的系统提示词

📎 [simonwillison.net/2026/Jul/2/dspy-datasette-agent-prompts](https://simonwillison.net/2026/Jul/2/dspy-datasette-agent-prompts/)

Simon 在 AIE Keynote 现场被 DSPy 触动，回来立刻把它接入自己的 Datasette Agent，用来做 SQL 系统提示词的自动评估与迭代。文章里附了完整的 Claude Code for web 会话记录——从需求描述、评估集构造，到 DSPy 编译出的最终 prompt，一步不省。

**为什么值得读**：这是当下"Prompt Engineering 从手写工艺走向可优化管线"最实操的公开案例之一。Simon 顺手写了配套的 [`llm-coding-agent 0.1a0`](https://simonwillison.net/2026/Jul/2/llm-coding-agent/)（用他的 llm 库 + Claude Fable 5 搭的极简 coding agent）和 [Understand to Participate](https://simonwillison.net/2026/Jul/2/understand-to-participate/)——后者一句话总结了 Geoffrey Litt 的观点："和 coding agent 协作时，你必须真正理解代码，否则你正在积累 cognitive debt。" 三篇连读，是本周 AI-coding 从业者最重要的自查框架。

### 2. 扎克伯格向员工承认：AI Agents 进展没到位

📎 [TechCrunch 报道](https://techcrunch.com/2026/07/02/mark-zuckerberg-tells-staff-that-ai-agents-havent-progressed-as-quickly-as-hed-hoped/) · [Meta 裁员反思](https://eshumarneedi.com/2026/07/03/zuckerberg-admits-metas-layoffs-were.html)（HN 199 分）

TechCrunch 拿到的 Meta 内部会议记录显示，扎克伯格罕见地对员工松口："AI agents 的开发进展没有达到我预想的速度。" 与此同时，另一篇 HN 高分文章《Zuckerberg 'Admits' Meta's Layoffs Were Ineffective》把去年那波"高效裁员论"反噬回来——两条对读，是 2026 下半年 Big Tech AI 叙事集体降温的最清晰信号。

**为什么值得读**：一个是资源最充沛、宣传声量最大的巨头，公开承认 agent 落地不如预期；另一个是"AI 时代裁员优化"叙事的公开翻车。这两条放在一起，比任何独立分析文都更能说明——2025 年"agent 元年"的乐观叙事，正在被真实产品化的复杂性反复打磨。做产品和做团队的人都该重新校准预期。

## 💬 社区热议 Community Buzz

### 🔥 "Half-Baked Product"（HN 1002 分）

📎 [weli.dev/blog/half-baked-product](https://weli.dev/blog/half-baked-product/)

作者提出一个尖锐的诊断：SaaS 领域最普遍的失败模式，是"半熟状态就上线"。文章给出了判断"半熟"的三条维度（核心用户旅程是否闭环、错误路径是否可恢复、跨设备一致性）与治理建议。评论区正反两派激烈：支持派认为"MVP 早已被曲解成半成品免死金牌"；反对派引用 Reid Hoffman 的"if you're not embarrassed by v1"回击。金句："The product isn't half-baked because you shipped early — it's half-baked because you never came back."

### 🔥 "Short Leash" AI 编程方法（HN 176 分，219 讨论）

📎 [blog.okturtles.org/2026/07/short-leash-ai-method](https://blog.okturtles.org/2026/07/short-leash-ai-method/)

作者提出与主流"给 agent 更多自主权"完全逆向的方法论：把每一步都压得极短、频繁 check-in，才是当前 SOTA 模型下的稳态用法。评论区分裂严重——从"这才是资深工程师的用法"到"你只是在用 chat 假装 agent"。219 条讨论说明工程师群体对 agent 使用模式的共识远未达成。

## ✍️ 今日寄语

> "You must truly understand the code, or you're accumulating cognitive debt."
> — Geoffrey Litt（via Simon Willison）

Agent 时代最容易的事，是让工具替你做完然后合上电脑。最难也最重要的事，是仍然愿意读懂它写下的每一行——因为债迟早要还，只是这次利息叫"你不知道自己在做什么"。
