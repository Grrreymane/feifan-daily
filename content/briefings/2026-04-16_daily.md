## 🔥 今日深度推荐 Deep Reads

### 1. Harness Engineering — OpenAI内部如何用0%人工代码运行百万行代码库

📎 [Latent Space Podcast](https://www.latent.space/podcast) · 嘉宾：Ryan Lopopolo (OpenAI Frontier)

OpenAI内部团队 Frontier 成了 Codex 最大的用户，运行超过100万行代码的代码库——令人震惊的是，**0%的代码由人类编写，甚至0%由人类review**。

Ryan Lopopolo 提出了「**Harness Engineering**」（线束工程）的概念：人类不再写代码，而是构建**约束框架（harness）**，让 AI Agent 在框架内自主完成从代码编写到 PR 合并的全流程。

核心洞察：
- 🔑 真正的瓶颈已从 token 转向**人类注意力**
- ⚡ 快速构建循环（1分钟上限）是关键
- 📋 Spec驱动开发 + 可观测性成为新范式
- 🤖 Symphony编排系统 + Skill蒸馏是内部核心工具

> **为什么值得读**：这是 AI 编程工作流最前沿的实践报告，直接来自 OpenAI 内部。"Harness Engineering" 可能成为继 "Vibe Coding" 和 "Context Engineering" 后的下一个行业术语。

### 2. Infold Games（无限暖暖/恋与深空开发商）：「AI缺乏灵魂，无法替代人类创作者」

📎 [GamesIndustry.biz](https://www.gamesindustry.biz/infinity-nikki-and-love-and-deepspace-dev-ai-lacks-the-soul-to-replace-human-creators)

在 AI 浪潮席卷游戏行业的当下，《无限暖暖》和《恋与深空》的开发商 Infold Games 公开承诺——**将人类创作者置于游戏开发的核心位置**。

这家中国游戏公司选择了与主流不同的路线，明确表态「AI 缺乏替代人类创造力的灵魂」。结合 GDC 2026 调查数据——**52%的游戏从业者认为生成式 AI 对行业产生负面影响**（去年30%，前年仅18%），视觉美术和技术美术从业者中这一比例更高达**64%**。

> **为什么值得读**：中国游戏公司对 AI 立场的罕见公开表态，且来自两款热门游戏的开发商。对于游戏美术从业者来说，这是「我们的价值不会被替代」的行业信号。

### 1. Claude Code Routines（⬆️690 💬389）

📎 [Claude Code Docs](https://code.claude.com/docs/en/routines) · [HN讨论](https://news.ycombinator.com/item?id=47768133)

Anthropic 发布 **Claude Code Routines** 功能，允许用户将编码工作流程自动化为**可重复执行的例程**。HN 上引发 690+ 赞和 389 条讨论。

**正方观点**：终于有了把 AI 编码从「一次性问答」升级为「可复用工作流」的工具，配合 Harness Engineering 理念，Agent 编程的基础设施正在快速成型。

**反方观点**：又一个「配置驱动开发」的工具，真正的编程难题不在于自动化重复流程，而在于处理边界情况和未知问题。还有人担忧这进一步加速了初级开发者技能退化。
