## 🔥 今日深度推荐 Deep Reads

### 1. Simon Willison — Who's Afraid of Chinese Models?

Simon Willison 昨日新文，回应 Ben Thompson 提出的"合法化蒸馏"提议，正面讨论美国开发者是否应在生产环境使用 DeepSeek、Qwen、Kimi、GLM 等中国开源权重模型。文章分三层展开：一是从技术角度看这些模型在下载量与实际部署上已全面反超美国开源阵营；二是"数据主权"与"模型来源"到底该怎样界定风险敞口——本地推理不联网的场景与调用中国厂商 API 的场景应严格区分；三是对个人开发者与企业 CTO 分别给出的选型建议。Simon 一如既往用具体例子说话，是本轮"AI 冷战叙事"里少数保持冷静的技术性判断，比空洞表态更有决策价值。

来源：[simonwillison.net](https://simonwillison.net/2026/Jul/20/whos-afraid-of-chinese-models/)

### 2. Charity Majors — AI Engineering as a Discipline

Honeycomb CTO Charity Majors 主张 "AI Engineering" 应被视为独立于 Software Engineering 和 ML Engineering 的第三个工程学科，并给出了具体的能力矩阵：Prompt 与上下文管理、评测体系、可观测性、成本工程、Agent 编排、失效模式设计等。她的核心论点是：传统 SWE 的确定性思维和 ML 工程的模型中心思维都不足以支撑生产级 AI 应用，我们需要一套新的心智模型来处理"输入不受控、输出非确定、失败模式无穷"的系统。对从传统研发转向 AI 应用工程的人，是一篇极好的方向感文章。

来源：[charity.wtf](https://charity.wtf/2026/07/ai-engineering-discipline)

## 💬 社区热议 Community Buzz

### HN 高赞：China's open-weights AI strategy is winning

[Hacker News 讨论](https://news.ycombinator.com/item?id=chinas-open-weights-winning)。这条讨论帖分析中国开源权重模型（DeepSeek、Qwen、Kimi、GLM）在 Hugging Face 下载量与实际部署量上已经全面超越美国开源模型。评论区呈两极：一派认为"开源赢家会成为事实标准"，援引 Android 与 Linux 类比；反方则担忧供应链信任问题，特别是权重是否可能植入后门、以及模型意识形态偏置的隐性成本。最精彩的一条评论把这场竞争概括为"美国用 API 卖模型，中国用权重塑生态"，把商业模式差异说得非常清楚。与 Simon Willison 今天的文章形成很好的互文。

### HN 重读经典：Corners Don't Look Like That — SSAO (2012)

[Nathan Reed 的经典博客](https://www.reedbeta.com/blog/ssao-corners)在 HN 被重新翻出。这篇 2012 年的老文讲清楚了 SSAO 为什么会在几何拐角处产生错误暗角，2026 年的 GTAO、HBAO+ 甚至部分 GI 方案仍然会踩到同样的坑。评论里现役图形程序员在讨论现代 Path Tracing 与 ReSTIR 是否已经彻底绕开这些问题，结论是"没有，只是把坑挪到别处"。TA 与图形程序员的长期必读。
