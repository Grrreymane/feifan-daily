## 🔥 今日深度推荐 Deep Reads

### 1. Martin Fowler 团队：怎么构建可靠的 Agentic AI 系统

**来源**：martinfowler.com｜HN 168 分 / 40 评论｜[原文](https://martinfowler.com/articles/reliable-llm-bayer.html)

这是 Bayer 内部和 Thoughtworks 合作的一线总结。文章不谈"AI 多神奇"，而是把 LLM Agent 当成一个**正常软件系统**来工程化：结构化输出、自检与 retry、可观测性、降级策略、契约测试，每一项都给出代码级范式。

文中最有价值的观点是：**当 agent 失败时，不要试图让 LLM "再聪明一点"，而是让外层架构假设它一定会失败**。这与昨天 Ian Barber 那篇《LLMs Are Complicated Now》构成完美延续——一个定义问题边界，一个给出工程化解法。

**为什么值得读**：所有在做 Knot 类 agent 平台、做 LLM 接入、做 AI 工作流的人，今天应该把这篇当作行业基准线来对照自己的系统。如果你做 LLM 应用还在"再写一版 prompt"层面，这篇会让你看到自己的差距在哪里。

### 2. 80 Level：手游城市级场景的生产、优化与世界观搭建

**来源**：80 Level｜[访谈页面](https://80.lv/articles/environment-art/)｜受访者 Bohdan Verteletskyi

国际游戏美术第一站 80 Level 这周放出的环境美术访谈，主题是"如何在手机性能预算内做出有可信度的城市级场景"。核心拆成四步：

1. **模块化资源 + 实例化**：先把"墙、窗、招牌、地砖"切片成 100 个左右的基础 kit；
2. **LOD 与流式加载**：以摄像机为中心的 ring buffer，远端只保留 silhouette；
3. **世界观一致性**：用 trim sheet 控制色彩与磨损度，让所有模块感觉来自同一个世界；
4. **AI 辅助贴图**：用 Substance 3D Sampler 把现实材质转成手游可用的小尺寸贴图，质量损失可控。

**为什么值得读**：这是手游 3D 美术 / 性能优化方向**少见的、完整的工程性访谈**。对正在做开放世界 / 大场景手游、或者准备从 PC 美术转手游美术的同事都有直接参考价值。
