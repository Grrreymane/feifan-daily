## 🔥 今日深度推荐

### 1. Simon Willison：Vibe Coding SwiftUI Apps 的快乐与反思

Simon Willison 新入手了一台 128GB M5 MacBook Pro，用来跑本地大模型效果极佳。他嫌 Activity Monitor 不好用，于是直接让 Claude 帮他 vibe coding 出了替代工具。文章详述了他的迭代流程：让 AI 写出初始版本，然后不断添加功能、不断打磨。

他引用了 Matt Webb 的观点："我现在看代码行数比以前少了，但思考架构比以前多了"——这正是 agentic coding 时代的核心转变。Agent 能把问题"碾碎"，但我们需要好的架构来确保代码可维护、可组合。

> 📎 [Simon Willison's Blog](https://simonwillison.net/2026/Mar/27/vibe-coding-swiftui/)

### 2. OpenAI 收购 Astral（uv / ruff / ty）—— Python 工具链巨变

OpenAI 正式收购了 Astral，也就是近两年爆火的 Python 工具链三件套——包管理器 `uv`、代码格式化器 `ruff`、以及类型检查器 `ty` 的开发团队。HN 1484 赞热帖。

社区反应两极分化：一边庆幸 Astral 拿到了资源加速开发，另一边担忧开源工具被大公司吞并后的命运——会不会重蹈 "拥抱→扩展→消灭" 的老路？

> 📎 [astral.sh](https://astral.sh)

## 💬 社区热议

### "Thoughts on slowing the fuck down" — HN 本周最热帖（1105 赞）

一篇呼吁在 AI 加速时代"慢下来"的文章引爆 Hacker News。作者的核心论点：工具在加速，但深度思考不该被工具驱动盲目前进。大量评论分享了个人经历——有人因为 AI 效率提升反而更焦虑，有人刻意减少工具依赖后感觉更好。

**正方**：AI 制造了一种"不进则退"的假象，实际上慢下来思考才是真正的竞争力。
**反方**：这是典型的"幸存者偏差"——能慢下来的人，本就已经在安全区。

> 📎 [mariozechner.at](https://mariozechner.at)

### GitHub 将默认使用私有仓库训练 AI（691 赞）

GitHub 新政策将默认把你的私有仓库用于训练 AI 模型，截止日期 4 月 24 日。大量开发者愤怒，讨论迁移到 Codeberg 等替代方案。还有人翻出了之前的隐私承诺截图——打脸速度令人咋舌。
