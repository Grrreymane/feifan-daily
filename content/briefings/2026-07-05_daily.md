## 🔥 今日深度推荐 Deep Reads

### ①《Agentic Coding Notes from Galapagos Island》— Dan Luu

- **原文**：<https://danluu.com/ai-coding/#appendix-agentic-loops-and-writing-this-post>
- **HN 讨论**：<https://news.ycombinator.com/item?id=48782671>

Dan Luu 用几周时间在真实项目里把 agentic coding 跑透了一遍，回来交出的是 2026 年最"务实"的一份长文——不是"AI 让我 10x 了"式吹捧，也不是"vibe coding 毁掉一代人"式挽歌，而是逐条列出：agent 在什么任务上真省时间、什么任务上会把项目搞爆、以及 agentic loop 该怎么设计才能把幻觉损失降到可接受。附录还记录了他"用 agent 写这篇文章本身"的元操作，observability 拉满，可复现。

**为什么值得读**：反鼓吹派必读。对每天在 Cursor / Claude Code / Fable 里做技术决策的人来说，这篇能替你省掉几十次踩坑。对做游戏工具链、TA pipeline 的人尤其有价值——他把 agent 应该在何处引入的判据说得比大部分厂商 blog 都清楚。

### ②《Midjourney 反诉好莱坞：请公开你们自家 AI 使用细节》

- **来源**：[TechCrunch](https://techcrunch.com/2026/07/04/midjourney-wants-hollywood-studios-to-reveal-the-details-of-their-ai-usage/)

迪士尼、环球对 Midjourney 的诉讼进入交叉揭露（discovery）阶段，MJ 反将一军，要求好莱坞公开自家生成 AI 使用细节。这是本轮 AI 版权战最戏剧性的一战——版权持有方本身也在悄悄用生成 AI，双标一旦被撕开，行业规则可能全面改写。

**为什么值得读**：对游戏美术尤其相关。你今天用 SD/MJ/Nano-Banana 做 concept 引用被大厂告，明天你甲方（发行/影视 IP 方）的内部 AI pipeline 也可能被同样标准审视。这场官司会决定 2027 年之后"AI 素材是否可用于商业交付"的边界。

## ✒️ 今日寄语

> "不要事无巨细地告诉模型怎么做，让它自己判断。"
> —— Cat Wu & Thariq Shihipar（Claude Code Team）

这句话反直觉，但 2026 年下半年最重要的工程转变正在发生：我们从"写完美 prompt"过渡到"设计足够好的判据 + 让 agent 自己走"。就像今天雪豹站在屋顶——你给它方向、给它光，剩下的它自己找角度。
