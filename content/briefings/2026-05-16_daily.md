## 🔥 今日深度推荐 Deep Reads

### 1. Simon Willison: "Not So Locked In Any More" — 编程语言锁定效应正在消失

📎 [simonwillison.net](https://simonwillison.net/2026/May/14/not-so-locked-in/#atom-everything)

Simon Willison 引用 Mitchell Hashimoto 关于 Bun 从 Zig 迁移到 Rust 的评论，展开了一场关于 AI 时代编程语言"可替代性"的深度思考。文中提到一家中型科技公司用 AI 编码 Agent **完成了 iPhone 和 Android 双端 app 的 React Native 完全重写**——这在以往几乎不可想象。

核心洞见：**"编程语言曾经是锁定，但现在不是了。"** 当 AI 能够以极低成本进行跨语言迁移时，技术栈选择的"沉没成本"将大幅降低。这对软件工程的根本假设构成颠覆——语言选择将从"战略决策"降格为"战术偏好"。

**为什么值得读**：如果你是技术从业者，这篇文章会改变你对技术栈选型的思考方式。Simon 一如既往地将宏观趋势浓缩为清晰的、可操作的洞见。

### 2. antirez: DS4 — 用 C 手写 DeepSeek V4 Flash 本地推理引擎

📎 [antirez.com/news/165](https://antirez.com/news/165) | [HN 讨论 (397票)](https://news.ycombinator.com/item?id=48142108)

Redis 之父 antirez 做了一件疯狂的事：用纯 C 语言从零手写了一个专为 DeepSeek V4 Flash 定制的本地推理引擎 `ds4.c`。这不是通用框架，而是针对单个模型的极致优化——涵盖模型加载、tokenizer、KV 缓存、Metal 计算图，一直到 OpenAI/Anthropic 兼容的 HTTP API。

更令人瞩目的是 antirez 同时分享的 Redis 新 Array 数据类型的 4 个月开发历程：**在 LLM 辅助下，他的开发效率获得了质的飞跃**。一个顶级 C 程序员对 AI 辅助编程的实战评价，可能比任何评测报告都更有说服力。

**为什么值得读**：这是"顶级黑客+AI辅助"的真实案例。代码质量极高，同时展示了 AI 如何改变即使是最资深开发者的工作方式。HN 上近 400 票的热度说明了一切。

## 💬 社区热议 Community Buzz

### 1. 拆除 RAV4 的蜂窝调制解调器和 GPS — 你的车在监视你吗？

📎 [原文](https://arkadiyt.com/2026/05/13/removing-the-modem-and-gps-from-my-rav4/) | [HN (1015票 · 534评论)](https://news.ycombinator.com/item?id=48138136)

**HN 今日最热帖**。一位车主详细记录了如何从 2024 款 Toyota RAV4 混动版中物理拆除蜂窝调制解调器和 GPS 模块。

🔥 **正方观点**：现代汽车默认收集大量位置和驾驶数据，车企可随时远程访问甚至锁车。用户应有权控制自己的设备。

⚖️ **反方观点**：拆除这些模块可能导致某些安全功能失效（如紧急呼叫 SOS）、影响保修，且随着 V2X 通信的普及可能违反未来法规。

💎 **精彩评论**："汽车行业正在复制手机行业的路径——先用便利诱惑你，再用数据变现榨干你。不同的是，手机你可以不买，车你必须得开。"

### 2. arXiv 新规：AI 幻觉参考文献将导致一年禁投

📎 [HN 讨论 (585票)](https://news.ycombinator.com/item?id=48140922)

arXiv 出台严厉新规：无论论文如何生成，作者必须对全部内容负责。若发现包含未经核实的 AI 生成幻觉参考文献，将被禁投一年。社区讨论焦点在于——学术界应该限制 AI 工具的使用，还是提高对作者的责任要求？
