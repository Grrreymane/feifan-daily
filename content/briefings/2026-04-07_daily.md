## 🔥 今日深度推荐 Deep Reads

### 1. 「八年的渴望，三个月的 AI 构建」— Lalit Maganti

🔗 [原文](https://lalitm.com/post/building-syntaqlite-ai/) ｜ HN ⬆️883 💬278

Lalit Maganti 花了八年构思一个"SQLite 应得的高保真开发工具"——syntaqlite，最终在三个月内借助 AI Agent 构建完成。这篇长文是近期最出色的 **Agentic Engineering 实践记录**之一。

**核心亮点：**
- 详细记录了如何将 AI Agent 作为合格的「工程同事」而非替代品——让 Agent 承担重复性代码编写，人类专注架构决策与质量审查
- 展示了正确使用 AI 的方法论：不是「vibe coding」式的放任，而是持续的人机协作循环
- Simon Willison 评价为"近期最好的 agentic engineering 长文"

**为什么值得读：** 这不是又一篇 "AI 帮我写了个 todo app" 的炫耀文。这是一个资深工程师用八年领域知识 + 三个月 AI 协作，构建了一个真正有深度的开发工具的故事。它回答了一个关键问题：AI 辅助开发的正确姿势到底是什么？

### 2. 「Vibe Coding 的邪教是疯狂的」— Bram Cohen（BitTorrent 创始人）

🔗 [原文](https://bramcohen.com/p/the-cult-of-vibe-coding-is-insane) ｜ HN ⬆️262 💬159

BitTorrent 发明者 Bram Cohen 犀利批评 Claude Code 团队的开发实践。Claude 源代码泄漏后，人们发现大量重复代码和混乱架构，正是因为团队坚持 vibe coding 原则——不看代码，只和 AI 对话。

**核心论点：**
- 纯 vibe coding 是个神话——你仍然在构建 plan files、skills、rules 等基础设施
- "Bad software is a decision you make"（烂软件是你主动选择的结果）
- 正确做法是：先审阅代码，发现问题，然后给 AI 高层次的概念性指导

和第一篇形成完美对照——一个展示 AI 协作的正面典型，一个批判极端 vibe coding 的反面教材。
