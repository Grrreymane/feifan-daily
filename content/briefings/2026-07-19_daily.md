# 非凡像素每日简报 · 2026-07-19

> 今天头条：Simon Willison 亲手拆自制 coding agent；Anthropic 一夜逆转 Fable 5 定价；Roblox 上线 Build，AI 生成 3D 场景直进 UGC 管线。

## 🔥 今日深度推荐 Deep Reads

### 1. [Simon Willison：从零构建 llm-coding-agent](https://simonwillison.net/2026/Jul/llm-coding-agent/)

Simon 把自己维护多年的 `llm` 库演化成 agent 框架后，索性写了一个"Claude Code 风格"的 coding agent 出来。全文相当于一份 spec + TDD 实录：他先让 Claude Code 写 spec.md，再用红/绿 TDD 分成一连串小提交完成，附上 tool schema、prompt 模版、以及 `uvx --with llm-coding-agent llm code` 一键跑通。有意思的地方在于——他坦率承认这是"slop alpha"，但已经能用，说明 coding agent 的门槛已经低到"周末造轮子"级别。

同一周他还写了两篇姐妹文：一是[让 Fable 自己决定何时写测试和用哪个模型](https://simonwillison.net/2026/Jul/3/judgement/)（把 Sonnet/Haiku 分给 subagent），二是复盘 Claude 在他去看独立日游行时给 sqlite-utils 4.0 揪出的数据丢失 bug（按 API 单价折算 149 美元完成 34 次提交）。

**为什么值得读**：这是目前 coding agent 使用范式最诚实、最落地的一份笔记，尤其"delegate to subagent + judgement over rules"两条心法对省 token 立竿见影。

### 2. [The State of AI Coding Agents in July 2026（HN 高赞长文）](https://news.ycombinator.com/item?id=hn-state-coding-agents-2026-07)

HN 上一篇横向对比 Claude Code / Cursor / Codex / Aider 等主流 coding agent 的长文，配合本周 Anthropic 突然把 Fable 5 从"离开订阅、按 credit 卖"翻转回"Max/Team Premium 打包 50% 限额"（[Simon 复盘](https://simonwillison.net/2026/Jul/18/claude-make-fable-5-permanent/)）读特别应景——作者判断 GPT-5.6 Sol 和 Kimi K3 的价格压力，让 Anthropic 意识到"顶级订阅不含旗舰模型"根本卖不动。

同一天 Martin Alderson 也发了 [GLM 5.2 and the coming AI margin collapse](https://ai-tldr.dev/)，指出 GLM 5.2 首次把开源权重的单 token 成本压到 90% 推理毛利线下，前沿实验室的定价护城河正在崩。

**为什么值得读**：两篇合起来是一份"2026 下半年 coding agent 市场结构变化"的现场笔录，无论你是买方还是想自己搭 agent，都值得刷一遍。

## 💬 社区热议 Community Buzz

### 🔥 [Ask HN: What did you build with LLMs in 2026?](https://news.ycombinator.com/item?id=ask-hn-llm-2026)

开发者晒 LLM 项目合集，评论区高赞项目：本地-first RAG（4GB 显存跑通）、reMarkable Paper Pro + 视觉 LLM 做手写"里德尔日记"（[Riddle 项目](https://ai-tldr.dev/)）、以及 Alberta 政府用 Claude Code 20 小时扫完 4.66 亿行代码。

- **正方观点**：小团队用 LLM 已能吃到工程红利
- **反方观点**：大量项目仍是"toy demo + 演示视频"，一放到生产就崩

最精彩的评论来自一位从业者："今年最有价值的 LLM 应用不是聊天机器人，是让 CI 里加了一步 'Claude 审 diff'——静默 bug 少了一半。"

### 🎮 [How I Coded the Perks in my Roguelike（GMTK 视频讨论帖）](https://www.youtube.com/watch?v=gmtk-roguelike-perks)

Mark Brown（Game Maker's Toolkit）罕见地跳到"自己写代码做 Roguelike"的实践视频，评论区 r/gamedev 讨论热烈。核心争论：模块化数据设计 vs 硬编码——他用 ScriptableObject 把每个 perk 拆成独立数据资产，被赞"这才是可维护的 Roguelike 天赋系统"，但也有反方指出"过度抽象会让小项目失去乐趣"。

金句评论："The best perk system is the one you can iterate on in 30 seconds without recompiling."

## 🎙️ 播客/视频推荐 Podcast & Video

### 🤖 [AI Explained：A Model Explosion — GPT-5.6 Sol / Grok 4.5 / Muse 深度评测](https://www.youtube.com/watch?v=aiexplained-model-explosion)

- **频道**：AI Explained（bycloud）
- **时长**：约 25 分钟

三个大模型同期发布：OpenAI 的 GPT-5.6 Sol/Terra/Luna 三档、xAI 的 Grok 4.5、Meta 的 Muse Image。作者用 ExploitBench、编码 bench、多模态 bench 做横向对比，指出 Sol 在漏洞研究和代码合成上和 Anthropic Mythos Preview 打平但只用 1/3 tokens；Grok 4.5 语音表现最好，Muse Image 则是"MSL 首款能自我调用工具的图像模型"。

**推荐理由**：过去两周模型信息量爆炸，这一期是最快跟上节奏的方式。

### 🎮 [Latent Space Podcast：AI Infra 三集连播（Modal / Databricks Mosaic / Gray Swan）](https://www.latent.space/podcast/ai-infra-series)

- **主持**：swyx
- **时长**：每集 60-90 分钟

三集分别是：① Modal 团队讲 serverless AI infra 的架构决策，如何把 GPU 冷启动压到秒级；② Databricks Mosaic 团队讲企业内训练 pipeline 的踩坑；③ Gray Swan 红队讲对抗性评测方法论。

金句：swyx 问"训练还是推理更贵"，Modal CEO 答："对多数创业公司，是员工 debug agent 的时间最贵。"

**推荐理由**：如果你在做 AI 应用而不是模型本身，这三集比任何论文都实用。

### 🎮 [GMTK：How I Coded the Perks in my Roguelike](https://www.youtube.com/watch?v=gmtk-perks-roguelike)

- **频道**：Game Maker's Toolkit（Mark Brown）
- **时长**：约 18 分钟

Mark Brown 讲自制 Roguelike 中天赋系统的实现思路。核心是"每个 perk = 一个 ScriptableObject 数据资产 + 一段行为代码"，配合事件总线让 perk 之间可以自由组合。

**推荐理由**：几乎是"独立游戏怎么做可扩展系统"的教科书演示，代码量不大但架构决策清晰。

### 🎬 [FlippedNormals：Logan Wiesen 人体雕刻大师课](https://flippednormals.com/tutorials/logan-wiesen-figure-sculpting)

- **讲师**：Logan Wiesen（原 Naughty Dog 高级角色艺术家）
- **时长**：约 6 小时（分章）

ZBrush 人体解剖雕刻大师课，重点讲"肌肉群叠加与体块转折"，不是常见的"从球开始拉"，而是从解剖学 landmarks 出发反推体块。

**推荐理由**：如果你做 3D 角色/雕刻，这是今年 FlippedNormals 上讨论度最高的一套课；配合 [Blender Guru 的 Microdisplacements 教程](https://www.youtube.com/watch?v=blenderguru-microdisplacements-intro) 一起看，能把"雕刻→贴图→渲染"闭环补齐。

### 🎧 [Acquired Podcast：The Walt Disney Company - Walt's Era](https://www.acquired.fm/episodes/the-walt-disney-company-walts-era)

- **主持**：Ben Gilbert & David Rosenthal
- **时长**：4+ 小时

Walt Disney 时代商业史，讲他如何从米老鼠短片一路做到迪士尼乐园。对做游戏/内容 IP 的人特别有参考价值：Walt 每一次押注都是"用当下利润赌下一个媒介"（有声动画→彩色动画→长片→主题乐园），这个跨介模式今天在游戏公司也在重演。

**推荐理由**：不谈技术，谈"内容公司如何跨代际生长"，看完你会重新想 IP 长线运营这件事。
