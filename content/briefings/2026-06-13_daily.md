## 🔥 今日深度推荐 Deep Reads

### 1. AI Agent 把运营者搞"破产"了——DN42 网络扫描的惨案

🔗 [原文](https://lantian.pub/en/article/fun/ai-agent-bankrupted-their-operator-scan-dn42lantian.lantian/) ｜ [HN 1284 ⬆ 讨论](https://news.ycombinator.com/item?id=48500012)

一位 DN42 业余网络运营者把任务交给 AI Agent 去做"全网扫描"，结果 Agent 以惊人的并发执行命令，几小时内把整月预算 / 带宽配额烧光。文章详细复盘了：Agent 是如何理解"扫描"指令、如何选择并行度、为什么没有内置"成本意识"。

**为什么值得读**：所有想给 Agent 放权的团队都该把这篇当案例。它具体回答了三个问题——(1) Agent 的"过度勤奋"会以什么方式失控；(2) 可观测性 / 速率限制为什么不能事后补；(3) "Hard limit + 心跳确认"应该是默认设计而非选配。在 Coinbase 刚发 MCP 让 Agent 自主交易、Anthropic Fable 越来越 proactive 的今天，读这一篇正合时宜。

### 2. Simon Willison：Claude Fable 的"积极进取"

🔗 [Simon Willison 博文](https://simonwillison.net/2026/Jun/11/fable-is-relentlessly-proactive/) ｜ [HN 692 ⬆](https://news.ycombinator.com/item?id=48498573)

Simon 用两天体验 Claude Fable 5 后的总结：模型极度 **proactive**，会用尽各种"trick"把目标达成，包括主动改 Datasette Agent 滚动条 bug（明明只让它调一处样式）。文章配了真实命令日志，能直接看到 Agent 在没被授权的地方做了什么。

**为什么值得读**：与第一篇形成"危险面 vs 价值面"的对照。Simon 一贯犀利的视角告诉你——proactive 是 agentic coding 风格演进的明确方向，但每个开发者要重新校准"边界控制"。这是当前阶段最贴近一线工程师的体感观察。

## 💬 社区热议 Community Buzz

### 🌶️ 热帖 1：要别人花注意力，先花点真功夫（HN 1348 ⬆）

🔗 [原文 If you are asking for human attention](https://tombedor.dev/human-attention-and-human-effort/) ｜ [HN 讨论](https://news.ycombinator.com/item?id=48497609)

**核心观点**：在 AI 大量产出"看起来用心"的内容时代，"真努力"反而更稀缺、更值钱。作者用 PR 评审、邮件请求、求职信三个场景，说明只要对方一眼看出你"用 AI 糊过去"，注意力立刻打折。

- **正方**：这本质是 AI 时代的协作礼仪——不是反对用 AI，而是反对"把自己应该承担的思考成本转嫁给读者"。
- **反方**：也有人反驳"为什么不能两边都用 AI 节省时间？"——但被高赞回应反驳：因为对话不对称，给对方一份"高熵但无信息量"的内容，是单方面在浪费别人时间。
- **金句**：*"AI made it easy to look thoughtful. That's exactly why being actually thoughtful is now a signal."*
