## 🔥 今日深度推荐 Deep Reads

### 1. Andrej Karpathy: Sequoia Ascent 2026 完整总结 — Software 3.0 思想体系

📎 [karpathy.bearblog.dev](https://karpathy.bearblog.dev/sequoia-ascent-2026/)

Karpathy 在 Sequoia 年度大会上发表了他对 Software 3.0 最完整的阐述。核心观点：**context window 是新的编程界面**。他将软件演进划分为三个时代——Software 1.0（手写代码）、2.0（神经网络权重）、3.0（用自然语言 prompt 编程）。最关键的洞察是 **"vibe coding 提升了地板，agentic engineering 提升了天花板"**——前者让非程序员也能构建软件，后者让专业工程师的生产力呈指数级增长。他还讨论了 AI 编程在过去两个月的质变——coding agents 在 2025 年 12 月前基本不可用，之后突然可用了，这不是渐进式改进，而是相变。

**为什么值得读**：这是 2026 年 AI 工程领域最重要的思想框架之一。Karpathy 用清晰的层次结构解释了当前 AI 编程革命的本质，对理解未来 2-3 年的技术走向有不可替代的参考价值。

### 2. Simon Willison: Sightings — 用 Claude Code 手机版在博客上集成 iNaturalist 野生动物照片

📎 [simonwillison.net](https://simonwillison.net/2026/May/2/sightings/#atom-everything)

Simon Willison 用一个极其优雅的实际案例展示了 AI 辅助编程的最新实践模式。他仅用三行简短指令，让 AI 智能体自主读取源码、完成数据库查询和前端界面修改，最后自动启动测试并提交了可直接合并的 Pull Request。整个过程中 agent 克隆了博客源码作为参考，自主学习项目结构和逻辑，避免了复杂 prompt 描述的需要。这展示了一种 **"克隆即文档"** 的 agentic engineering 新范式。

**为什么值得读**：这不是理论讨论，而是一个活生生的、可立即复制的工作流。Simon 是业内公认最擅长将前沿 AI 能力转化为日常生产力的实践者。

## 💬 社区热议 Community Buzz

### 1. VS Code 在提交中强制插入 "Co-Authored-by Copilot" 引发社区愤怒

📎 [GitHub PR #310226](https://github.com/microsoft/vscode/pull/310226) | [HN 讨论 (1416⬆️)](https://news.ycombinator.com/item?id=47989883)

VS Code 的一个 PR 引爆了社区：微软在 git commit 信息中**自动插入 "Co-Authored-by: GitHub Copilot"**，无论开发者是否实际使用了 Copilot。HN 上 1400+ 赞，讨论极其激烈。

**正方观点**：这是透明度的体现，让人知道 AI 参与了代码编写。
**反方观点（占压倒性多数）**：这是强行在用户的 commit history 里打广告；违反了开发者对自己代码归属权的基本尊重；更荒谬的是即使完全没用 Copilot 也会被标注。

**最精彩评论**："This is the Git equivalent of a company putting their logo on your personal art after you used their pencil."
