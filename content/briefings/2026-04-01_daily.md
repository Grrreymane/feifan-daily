## 🔥 今日深度推荐 Deep Reads

### 1. Ollama 接入 Apple MLX 框架 —— Mac 本地 AI 推理迎来里程碑飞跃

📎 [Ollama 官方博客](https://ollama.com/blog/mlx) | [MacRumors 报道](https://www.macrumors.com/2026/03/31/ollama-now-runs-faster-apple-silicon-macs/) | HN 560⬆️ 282💬

Ollama 0.19 预览版正式整合 Apple MLX 框架，充分利用 Apple Silicon 统一内存架构带来重大性能提升：**预填充速度提升 1.6 倍、解码速度接近翻倍**。在 M5 芯片上利用 GPU Neural Accelerators 获得最大加速效果。

新版还引入了 NVFP4 量化格式和改进的 KV 缓存管理，当前支持 Qwen3.5-35B-A3B 模型，需 32GB+ 统一内存。这意味着 Mac 用户终于可以在本地高效运行 35B 参数级别的模型了。

**为什么值得读**：这是本地 AI 推理领域的里程碑事件。如果你有一台 M 系列芯片的 Mac，你的本地 LLM 体验即将发生质变。Apple Silicon 的统一内存架构正在成为本地 AI 推理的最大优势——连 NVIDIA 都没有这种 CPU/GPU 共享内存的便利。

### 2. Claude Code 源码泄露 —— AI 编码工具的安全隐患与内部运作机制曝光

📎 [原帖](https://twitter.com/Fried_rice/status/2038894956459290963) | [HN 讨论](https://news.ycombinator.com/item?id=47584540) | HN 1322⬆️ 691💬

Anthropic 的 Claude Code 工具的完整源代码通过 NPM registry 中的 source map 文件意外泄露。泄露内容包括：完整的系统提示词（system prompt）、所有工具定义、内部调用链路等核心实现细节。

社区对此进行了爆炸性讨论，关注点覆盖：① AI 编程工具的安全性底线在哪里？② NPM 发布流程的安全隐患——source map 文件意外泄露已不是第一次；③ Claude Code 的内部工作原理暴露后，对竞品意味着什么？

**为什么值得读**：这不只是一个安全事故——它让我们第一次窥见了 AI 编码助手真正的内部运作方式。那些精心设计的系统提示词、工具调用策略和上下文管理逻辑，是 AI 编程工具真正的"核心IP"。对于任何使用或构建 AI 工具的开发者来说，这是一次难得的拆机课。

### 1. Axios NPM 供应链攻击 —— 朝鲜黑客瞄准开源生态命脉

📎 [StepSecurity 分析](https://www.stepsecurity.io/blog/axios-compromised-on-npm-malicious-versions-drop-remote-access-trojan) | [TechCrunch](https://techcrunch.com/2026/03/31/hacker-hijacks-axios-open-source-project-used-by-millions-to-push-malware/) | HN 1641⬆️ 645💬

每周下载量数千万次的热门 HTTP 库 Axios 在 NPM 上被入侵，恶意版本植入远程访问木马(RAT)，据报道与朝鲜黑客有关。这是继 event-stream、ua-parser-js 之后又一起重大 NPM 供应链攻击。

🔴 **正方**：暴露了开源供应链安全的系统性风险，单点失败就能波及数百万项目。呼吁更严格的包签名和发布审计机制。

🔵 **反方**：开源的开放性本身就是安全风险与创新活力的权衡。过度管控会扼杀贡献者的积极性。
