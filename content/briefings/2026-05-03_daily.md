## 🔥 今日深度推荐 Deep Reads

### 1. Simon Willison：在手机上用 Claude Code 构建 iNaturalist 集成

📎 [simonwillison.net](https://simonwillison.net/2026/May/2/sightings/#atom-everything)

Simon 这两天在露营途中用手机完成了一整套功能开发——用 Claude Code for web 在手机上构建了一个 iNaturalist 野生动物观察数据与博客的集成系统。

**核心亮点：**
- 📱 **纯手机开发**：在露营途中没有笔记本，用 Claude Code 的 web 版在手机上完成了从 Python CLI 工具到前端展示的全部开发
- 🦊 回填了十多年的 iNaturalist 观察数据——搜索 "lemur" 就能看到他 2019 年在马达加斯加拍的狐猴照片
- 构建了一个 "clumper" 工具，将时间和距离接近的观察自动分组
- 作为他 "beats system" 外部内容聚合体系的延伸，观察记录会出现在首页、日期归档和站内搜索中

Simon 已成为 AI 原生开发的活百科全书。这篇文章不仅展示了移动端 AI 编程的真实可行性（"我现在 95% 的代码不是自己打的，大量代码都在手机上写"），更体现了一种"生活即创作"的工程师哲学。

### 2. Zig 项目坚定的反 AI 贡献政策——以及背后的「Contributor Poker」逻辑

📎 [simonwillison.net 引用](https://simonwillison.net/2026/Apr/30/)

Zig 是目前最严格的反 LLM 开源项目之一：**禁止 LLM 用于 issues、pull requests 和 bug tracker 评论**。创始人 Andrew Kelley 的核心论点：

> "用 LLM 的人和不用的人之间的区别，就像吸烟者走进房间——每个不吸烟的人都能立刻闻到。我不是叫你别抽，但请别在我家抽。"

更精彩的是 Loris 的 "Contributor Poker" 概念——"就像真正的扑克一样——你赌的是人，不是牌。"

这是当前开源社区关于 AI 贡献最深刻的讨论之一。它从维护者稀缺注意力的经济学角度出发——如果一个 PR 主要是 LLM 写的，维护者为什么要花时间审查，而不是自己开个 LLM 来解决同样的问题？
